# 日志组件 (Log)

## 概述

日志组件是 Due 框架的核心模块之一，提供结构化、高性能的日志记录功能。支持多种日志级别、自定义格式化、文件轮转等特性。

## 包路径

```go
import "github.com/dobyte/due/v2/log"
```

## 日志级别

Due 框架支持以下日志级别（按优先级从低到高）：

- `DebugLevel` - 调试级别，用于开发调试信息
- `InfoLevel` - 信息级别，用于一般信息记录
- `WarnLevel` - 警告级别，用于潜在问题提示
- `ErrorLevel` - 错误级别，用于错误信息记录
- `FatalLevel` - 致命级别，记录后程序会退出
- `PanicLevel` - 恐慌级别，记录后程序会 panic

## 配置参数

在 `etc.toml` 配置文件中配置日志参数：

```toml
[log]
    # 日志输出级别，可选：debug | info | warn | error | fatal | panic
    level = "info"
    # 堆栈的最低输出级别，可选：debug | info | warn | error | fatal | panic
    stackLevel = "error"
    # 时间格式，标准库时间格式
    timeFormat = "2006/01/02 15:04:05.000000"
    # 输出栈的跳过深度
    callSkip = 2
    # 是否启用调用文件全路径
    callFullPath = false
    # 日志输出终端
    terminals = ["console", "file"]
    
    # 控制台同步器配置
    [log.console]
        # 日志输出格式，可选：text | json
        format = "text"
    
    # 文件同步器配置
    [log.file]
        # 输出文件路径
        path = "./log/due.log"
        # 日志输出格式，可选：text | json
        format = "text"
        # 文件最大留存时间，d:天、h:时、m:分、s:秒
        maxAge = "7d"
        # 文件最大尺寸限制，支持单位： B | K | KB | M | MB | G | GB | T | TB | P | PB | E | EB | Z | ZB，默认为100M
        maxSize = "100M"
        # 文件翻转方式，可选：none | year | month | week | day | hour，默认为none
        rotate = "none"
        # 文件翻转时是否对文件进行压缩
        compress = false
```

## 核心 API

### 基础日志函数

```go
// Debug 记录调试日志
func Debug(args ...interface{})

// Info 记录信息日志
func Info(args ...interface{})

// Warn 记录警告日志
func Warn(args ...interface{})

// Error 记录错误日志
func Error(args ...interface{})

// Fatal 记录致命错误日志并退出程序
func Fatal(args ...interface{})

// Panic 记录恐慌日志并 panic
func Panic(args ...interface{})
```

### 格式化日志函数

```go
// Debugf 记录格式化的调试日志
func Debugf(format string, args ...interface{})

// Infof 记录格式化的信息日志
func Infof(format string, args ...interface{})

// Warnf 记录格式化的警告日志
func Warnf(format string, args ...interface{})

// Errorf 记录格式化的错误日志
func Errorf(format string, args ...interface{})

// Fatalf 记录格式化的致命错误日志并退出程序
func Fatalf(format string, args ...interface{})

// Panicf 记录格式化的恐慌日志并 panic
func Panicf(format string, args ...interface{})
```

### 结构化日志

```go
// WithField 添加单个字段
func WithField(key string, value interface{}) Logger

// WithFields 添加多个字段
func WithFields(fields ...Field) Logger

// WithError 添加错误字段
func WithError(err error) Logger
```

## 使用示例

### 基础使用

```go
package main

import "github.com/dobyte/due/v2/log"

func main() {
    // 基础日志
    log.Debug("debug message")
    log.Info("info message")
    log.Warn("warn message")
    log.Error("error message")
    
    // 格式化日志
    log.Infof("user %s logged in", "admin")
    log.Errorf("failed to connect: %v", err)
    
    // 结构化日志
    log.WithField("user_id", 123).Info("user action")
    log.WithFields(
        log.Field{Key: "user_id", Value: 123},
        log.Field{Key: "action", Value: "login"},
    ).Info("user action")
    
    // 带错误的日志
    if err := doSomething(); err != nil {
        log.WithError(err).Error("operation failed")
    }
}
```

### 自定义日志实例

```go
package main

import "github.com/dobyte/due/v2/log"

func main() {
    // 创建自定义日志实例
    logger := log.New(
        log.WithLevel(log.DebugLevel),
        log.WithCallerSkip(1),
    )
    
    logger.Info("custom logger message")
}
```

## 最佳实践

1. **选择合适的日志级别**：
   - 开发调试信息使用 `Debug`
   - 一般业务信息使用 `Info`
   - 潜在问题使用 `Warn`
   - 错误情况使用 `Error`

2. **使用结构化日志**：
   - 使用 `WithField` 和 `WithFields` 添加上下文信息
   - 避免在日志消息中拼接变量

3. **生产环境配置**：
   - 设置合适的日志级别（建议 `info` 或 `warn`）
   - 启用文件输出并配置轮转策略
   - 设置 `callFullPath = false` 减少日志体积

4. **性能优化**：
   - 在高并发场景下使用异步日志
   - 避免在循环中记录大量日志
   - 使用 `JSON` 格式便于日志收集和分析
