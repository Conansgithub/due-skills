# 分布式锁 (Lock)

## 概述

分布式锁模块是 Due 框架的扩展组件,提供分布式环境下的互斥访问能力。支持 Redis、Etcd 等存储后端,确保多节点环境下的数据一致性。

## 包路径

```go
import "github.com/dobyte/due/lock/redis/v2"
```

## Redis 分布式锁

### 配置参数

```toml
[lock.redis]
    # 客户端连接地址
    addrs = ["127.0.0.1:6379"]
    # 数据库号
    db = 0
    # 用户名
    username = ""
    # 密码
    password = ""
    # 私钥文件
    keyFile = ""
    # 证书文件
    certFile = ""
    # CA证书文件
    caFile = ""
    # 最大重试次数
    maxRetries = 3
    # key前缀
    prefix = "due:lock"
```

### 使用示例

```go
package main

import (
    "github.com/dobyte/due/lock/redis/v2"
    "github.com/dobyte/due/v2"
)

func main() {
    // 创建容器
    container := due.NewContainer()
    // 创建分布式锁
    locker := redis.NewLock()
    // 添加到容器
    container.Add(locker)
    // 启动容器
    container.Serve()
}
```

## 核心 API

### 锁接口

```go
type Lock interface {
    // 获取锁
    Acquire(ctx context.Context, key string, ttl time.Duration) (Locker, error)
    // 尝试获取锁
    TryAcquire(ctx context.Context, key string, ttl time.Duration) (Locker, bool, error)
}
```

### 锁实例接口

```go
type Locker interface {
    // 释放锁
    Release(ctx context.Context) error
    // 续期锁
    Renew(ctx context.Context, ttl time.Duration) error
}
```

## 使用场景示例

### 基础使用

```go
package main

import (
    "context"
    "time"
    "github.com/dobyte/due/lock/redis/v2"
)

func main() {
    // 创建锁
    locker := redis.NewLock()
    
    ctx := context.Background()
    
    // 获取锁
    lock, err := locker.Acquire(ctx, "my-resource", 10*time.Second)
    if err != nil {
        panic(err)
    }
    defer lock.Release(ctx)
    
    // 执行业务逻辑
    doSomething()
}
```

### 尝试获取锁

```go
package main

import (
    "context"
    "time"
    "github.com/dobyte/due/lock/redis/v2"
)

func main() {
    // 创建锁
    locker := redis.NewLock()
    
    ctx := context.Background()
    
    // 尝试获取锁
    lock, acquired, err := locker.TryAcquire(ctx, "my-resource", 10*time.Second)
    if err != nil {
        panic(err)
    }
    
    if !acquired {
        println("Failed to acquire lock")
        return
    }
    defer lock.Release(ctx)
    
    // 执行业务逻辑
    doSomething()
}
```

### 锁续期

```go
package main

import (
    "context"
    "time"
    "github.com/dobyte/due/lock/redis/v2"
)

func main() {
    // 创建锁
    locker := redis.NewLock()
    
    ctx := context.Background()
    
    // 获取锁
    lock, err := locker.Acquire(ctx, "my-resource", 10*time.Second)
    if err != nil {
        panic(err)
    }
    defer lock.Release(ctx)
    
    // 执行业务逻辑
    go func() {
        time.Sleep(5 * time.Second)
        // 续期锁
        err := lock.Renew(ctx, 10*time.Second)
        if err != nil {
            panic(err)
        }
    }()
    
    // 长时间任务
    doLongRunningTask()
}
```

## 最佳实践

1. **超时设置**:
   - 设置合理的锁超时时间防止死锁
   - 超时时间应大于业务执行时间

2. **锁续期**:
   - 对于长时间任务,使用锁续期机制
   - 避免锁过期导致并发问题

3. **错误处理**:
   - 处理获取锁失败的情况
   - 实现重试机制提高可用性

4. **资源释放**:
   - 使用 `defer` 确保锁被释放
   - 处理释放锁时的错误

5. **Key命名**:
   - 使用清晰的key前缀避免冲突
   - 包含业务标识便于调试
