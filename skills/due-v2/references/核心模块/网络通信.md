# 网络通信 (Network)

## 概述

网络通信模块是 Due 框架的核心组件,提供高性能的 TCP 和 WebSocket 网络通信能力。支持自定义协议、心跳检测、TLS 加密等特性。

## 包路径

```go
import (
    "github.com/dobyte/due/network/tcp/v2"
    "github.com/dobyte/due/network/ws/v2"
)
```

## TCP 网络

### 配置参数

```toml
[network.tcp.server]
    # 服务器监听地址
    addr = ":3553"
    # 最大连接数,默认为10000
    maxConnNum = 10000
    # 拨号超时时间,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为5s
    timeout = "5s"
    # 私钥文件
    keyFile = ""
    # 证书文件
    certFile = ""
    # CA证书文件
    caFile = ""
    # 心跳间隔时间;设置为0则不启用心跳检测,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为10s
    heartbeatInterval = "10s"

[network.tcp.client]
    # 拨号地址
    addr = "127.0.0.1:3553"
    # 拨号超时时间,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为5s
    timeout = "5s"
    # CA证书文件
    caFile = ""
    # 证书域名
    serverName = ""
    # 心跳间隔时间;设置为0则不启用心跳检测,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为10s
    heartbeatInterval = "10s"
```

### TCP 服务器示例

```go
package main

import (
    "github.com/dobyte/due/network/tcp/v2"
    "github.com/dobyte/due/v2"
    "github.com/dobyte/due/v2/log"
)

func main() {
    // 创建容器
    container := due.NewContainer()
    // 创建TCP服务器
    server := tcp.NewServer()
    // 添加网络组件
    container.Add(server)
    // 启动容器
    container.Serve()
}
```

## WebSocket 网络

### 配置参数

```toml
[network.ws.server]
    # 服务器监听地址
    addr = ":3553"
    # 最大连接数,默认为10000
    maxConnNum = 10000
    # 私钥文件

    certFile = ""
    # CA证书文件
    caFile = ""

    handshakeTimeout = "10s"
    # 心跳间隔时间;设置为0则不启用心跳检测,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为10s

[network.ws.client]
    # 拨号地址
    url = "ws://127.0.0.1:3553"
    # 握手超时时间,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为10s

    # 心跳间隔时间;设置为0则不启用心跳检测,支持单位:纳秒(ns)、微秒(us | µs)、毫秒(ms)、秒(s)、分(m)、小时(h)、天(d)。默认为10s
```

## 消息协议配置

```toml
[packet]
    # 字节序,默认为big。可选:little | big
    byteOrder = "big"
    # 路由字节数,默认为2字节
    routeBytes = 2
    # 序列号字节数,默认为2字节
    seqBytes = 2
    # 消息字节数,默认为5000字节
    bufferBytes = 5000
    # 是否携带服务器心跳时间
    heartbeatTime = false
```

## 最佳实践

1. **连接管理**:
   - 合理设置 `maxConnNum` 防止资源耗尽
   - 启用心跳检测及时清理死连接

2. **安全性**:
   - 生产环境启用 TLS 加密
   - 配置证书文件确保通信安全

3. **性能优化**:
   - 根据业务需求调整消息缓冲区大小
   - 合理设置超时时间避免阻塞

4. **协议设计**:
   - 统一使用大端字节序保证跨平台兼容性
   - 合理设计路由号避免冲突
