# 配置中心 (Config)

## 概述

配置中心是 Due 框架的核心组件，提供统一的配置管理能力。支持本地文件、环境变量、远程配置中心（Consul、Etcd、Nacos）等多种配置源，实现配置的动态更新和热加载。

## 包路径

```go
import "github.com/dobyte/due/v2/etc"
```

## 本地文件配置

### 配置文件格式

Due 框架支持以下配置文件格式：
- TOML (etc.toml)
- JSON (etc.json)
- YAML (etc.yaml, etc.yml)

### 配置示例

```toml
# 进程号
pid = "./run/due.pid"
# 开发模式。支持模式：debug、test、release
mode = "debug"
# 统一时区设置
timezone = "Local"

[cluster.node]
    # 实例ID
    id = ""
    # 实例名称
    name = "node"
    # 内建RPC服务器监听地址
    addr = ":0"
```

## 核心 API

### 配置读取

```go
package etc

// 获取配置值
func Get(key string, def ...interface{}) Value

// 检查配置是否存在
func Has(key string) bool

// 设置配置值
func Set(key string, value interface{})

// 获取所有配置
func All() map[string]interface{}
```

### Value 接口

```go
type Value interface {
    // 转换为字符串
    String() string
    // 转换为整数
    Int() int
    // 转换为64位整数
    Int64() int64
    // 转换为浮点数
    Float64() float64
    // 转换为布尔值
    Bool() bool
    // 转换为时间
    Duration() time.Duration
    // 转换为字符串切片
    Strings() []string
    // 转换为接口切片
    Interfaces() []interface{}
    // 转换为map
    Map() map[string]interface{}
}
```

## 使用场景示例

### 基础使用

```go
package main

import (
    "github.com/dobyte/due/v2/etc"
)

func main() {
    // 读取字符串配置
    name := etc.Get("cluster.node.name").String()
    println("Node name:", name)
    
    // 读取整数配置
    port := etc.Get("cluster.node.port", 8080).Int()
    println("Port:", port)
    
    // 读取布尔配置
    debug := etc.Get("debug", false).Bool()
    println("Debug mode:", debug)
    
    // 读取时间配置
    timeout := etc.Get("timeout", "5s").Duration()
    println("Timeout:", timeout)
    
    // 检查配置是否存在
    if etc.Has("cluster.node.id") {
        id := etc.Get("cluster.node.id").String()
        println("Node ID:", id)
    }
}
```

### 嵌套配置读取

```go
package main

import (
    "github.com/dobyte/due/v2/etc"
)

func main() {
    // 读取嵌套配置
    addrs := etc.Get("locate.redis.addrs").Strings()
    for _, addr := range addrs {
        println("Redis address:", addr)
    }
    
    // 读取map配置
    metadata := etc.Get("cluster.node.metadata").Map()
    for k, v := range metadata {
        println(k, ":", v)
    }
}
```

### 动态配置更新

```go
package main

import (
    "github.com/dobyte/due/v2/etc"
)

func main() {
    // 设置配置值
    etc.Set("cluster.node.name", "node-1")
    
    // 读取更新后的配置
    name := etc.Get("cluster.node.name").String()
    println("Updated node name:", name)
}
```

## 环境变量配置

### 使用示例

## 远程配置中心

### Consul 配置中心

```toml
[config.consul]
    # 客户端连接地址
    addr = "127.0.0.1:8500"
    # 配置路径
    path = "config/due"
    # 是否启用监听
    watch = true
```

```go
package main

import (
    "github.com/dobyte/due/config/consul/v2"
    "github.com/dobyte/due/v2"
)

func main() {
    // 创建容器
    container := due.NewContainer()
    // 创建配置中心
    config := consul.NewConfig()
    // 添加到容器
    container.Add(config)
    // 启动容器
    container.Serve()
}
```

## 最佳实践

1. **配置分层**:
   - 使用点号分隔的key组织配置结构
   - 例如：`cluster.node.name`、`cluster.node.addr`

2. **默认值**:
   - 为重要配置设置合理的默认值
   - 使用 `etc.Get(key, defaultValue)` 提供默认值

3. **配置验证**:
   - 在启动时验证必要配置是否存在
   - 检查配置值的合法性

4. **热更新**:
   - 使用远程配置中心实现配置热更新
   - 监听配置变更事件及时响应

5. **安全性**:
   - 敏感配置使用环境变量或密钥管理服务
   - 不要将密码等敏感信息硬编码在配置文件中
