# 事件总线 (EventBus)

## 概述

事件总线是 Due 框架的核心组件，提供发布-订阅模式的事件通信能力。支持同步和异步事件处理，实现模块间的松耦合通信。

## 包路径

```go
import "github.com/dobyte/due/v2/core/eventbus"
```

## 核心 API

### 事件总线接口

```go
type EventBus interface {
    // 订阅事件
    Subscribe(event string, handler EventHandler)
    // 取消订阅
    Unsubscribe(event string, handler EventHandler)
    // 发布事件
    Publish(event string, data interface{})
    // 异步发布事件
    PublishAsync(event string, data interface{})
}
```

### 事件处理器

```go
type EventHandler func(event string, data interface{})
```

## 使用场景示例

### 基础使用

```go
package main

import (
    "github.com/dobyte/due/v2/core/eventbus"
)

func main() {
    // 获取全局事件总线
    bus := eventbus.GlobalEventBus
    
    // 订阅事件
    bus.Subscribe("user.login", func(event string, data interface{}) {
        userID := data.(int64)
        println("User logged in:", userID)
    })
    
    // 发布事件
    bus.Publish("user.login", int64(1001))
}
```

### 多个订阅者

```go
package main

import (
    "github.com/dobyte/due/v2/core/eventbus"
)

func main() {
    bus := eventbus.GlobalEventBus
    
    // 订阅者1
    bus.Subscribe("user.register", func(event string, data interface{}) {
        println("Subscriber 1: User registered")
    })
    
    // 订阅者2
    bus.Subscribe("user.register", func(event string, data interface{}) {
        println("Subscriber 2: User registered")
    })
    
    // 发布事件，所有订阅者都会收到
    bus.Publish("user.register", nil)
}
```

### 异步事件

```go
package main

import (
    "time"
    "github.com/dobyte/due/v2/core/eventbus"
)

func main() {
    bus := eventbus.GlobalEventBus
    
    // 订阅异步事件
    bus.Subscribe("user.action", func(event string, data interface{}) {
        time.Sleep(1 * time.Second)
        println("Async event processed")
    })
    
    // 异步发布事件，不阻塞
    bus.PublishAsync("user.action", nil)
    
    println("Event published asynchronously")
    
    // 等待异步事件处理完成
    time.Sleep(2 * time.Second)
}
```

### 取消订阅

```go
package main

import (
    "github.com/dobyte/due/v2/core/eventbus"
)

func main() {
    bus := eventbus.GlobalEventBus
    
    // 定义处理器
    handler := func(event string, data interface{}) {
        println("Event received")
    }
    
    // 订阅事件
    bus.Subscribe("user.logout", handler)
    
    // 发布事件
    bus.Publish("user.logout", nil)
    
    // 取消订阅
    bus.Unsubscribe("user.logout", handler)
    
    // 再次发布事件，处理器不会被调用
    bus.Publish("user.logout", nil)
}
```

### 在节点服务器中使用

```go
package main

import (
    "github.com/dobyte/due/v2/cluster/node"
    "github.com/dobyte/due/v2/core/eventbus"
)

func main() {
    // 创建节点组件
    component := node.NewNode()
    
    // 获取节点代理
    proxy := component.Proxy()
    
    // 监听节点启动事件
    proxy.AddHookListener(node.Start, func(proxy *node.Proxy) {
        println("Node started")
        
        // 发布自定义事件
        eventbus.GlobalEventBus.Publish("node.ready", nil)
    })
    
    // 订阅自定义事件
    eventbus.GlobalEventBus.Subscribe("node.ready", func(event string, data interface{}) {
        println("Node is ready")
    })
}
```

### 在网关服务器中使用

```go
package main

import (
    "github.com/dobyte/due/v2/cluster/gate"
    "github.com/dobyte/due/v2/core/eventbus"
)

func main() {
    // 创建网关组件
    component := gate.NewGate()
    
    // 获取网关代理
    proxy := component.Proxy()
    
    // 监听连接事件
    proxy.AddEventListener(gate.Connect, func(conn *gate.Conn) {
        // 发布连接事件
        eventbus.GlobalEventBus.Publish("client.connect", conn.ID())
    })
    
    // 监听断开连接事件
    proxy.AddEventListener(gate.Disconnect, func(conn *gate.Conn) {
        // 发布断开事件
        eventbus.GlobalEventBus.Publish("client.disconnect", conn.ID())
    })
    
    // 订阅连接事件
    eventbus.GlobalEventBus.Subscribe("client.connect", func(event string, data interface{}) {
        connID := data.(int64)
        println("Client connected:", connID)
    })
}
```

## 事件类型

### 系统事件

Due 框架预定义了一些系统事件：

```go
const (
    // 节点启动
    Start = "start"
    // 节点停止
    Stop = "stop"
    // 连接建立
    Connect = "connect"
    // 连接断开
    Disconnect = "disconnect"
)
```

### 自定义事件

```go
package main

import (
    "github.com/dobyte/due/v2/core/eventbus"
)

const (
    UserLogin    = "user.login"
    UserLogout   = "user.logout"
    UserRegister = "user.register"
)

func main() {
    bus := eventbus.GlobalEventBus
    
    // 订阅自定义事件
    bus.Subscribe(UserLogin, handleUserLogin)
    bus.Subscribe(UserLogout, handleUserLogout)
    bus.Subscribe(UserRegister, handleUserRegister)
    
    // 发布自定义事件
    bus.Publish(UserLogin, map[string]interface{}{
        "user_id": 1001,
        "username": "alice",
    })
}

func handleUserLogin(event string, data interface{}) {
    userInfo := data.(map[string]interface{})
    println("User logged in:", userInfo["username"].(string))
}

func handleUserLogout(event string, data interface{}) {
    println("User logged out")
}

func handleUserRegister(event string, data interface{}) {
    println("User registered")
}
```

## 最佳实践

1. **事件命名**:
   - 使用清晰的命名规范，如 `module.action`
   - 例如：`user.login`、`order.create`

2. **异步处理**:
   - 对于耗时操作使用异步事件
   - 避免阻塞主业务流程

3. **错误处理**:
   - 在事件处理器中处理错误
   - 不要让错误影响其他订阅者

4. **资源管理**:
   - 及时取消不需要的订阅
   - 避免内存泄漏

5. **线程安全**:
   - 事件处理器要考虑并发安全
   - 使用锁或原子操作保护共享数据

6. **性能优化**:
   - 避免在事件处理器中执行重操作
   - 合理使用异步事件提高吞吐量
