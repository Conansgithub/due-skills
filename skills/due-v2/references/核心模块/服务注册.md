# 服务注册 (Registry)

## 概述

服务注册模块是 Due 框架的核心组件,提供服务发现、服务注册、负载均衡等分布式系统基础能力。支持 Consul、Etcd、Nacos 等主流注册中心。

## 包路径

```go
import "github.com/dobyte/due/registry/consul/v2"
```

## Consul 注册中心

### 配置参数

```toml
[registry.consul]
    # 客户端连接地址,默认为127.0.0.1:8500
    addr = "127.0.0.1:8500"
    # 是否启用健康检查,默认为true
    healthCheck = true
    # 健康检查时间间隔(秒),仅在启用健康检查后生效,默认为10
    healthCheckInterval = 10
    # 健康检查超时时间(秒),仅在启用健康检查后生效,默认为5
    healthCheckTimeout = 5
    # 是否启用心跳检查,默认为true
    heartbeatCheck = true
    # 心跳检查时间间隔(秒),仅在启用心跳检查后生效,默认为10
    heartbeatCheckInterval = 10
    # 健康检测失败后自动注销服务时间(秒),默认为30
    deregisterCriticalServiceAfter = 30
```

### 使用示例

```go
package main

import (
    "github.com/dobyte/due/registry/consul/v2"
    "github.com/dobyte/due/v2"
)

func main() {
    // 创建容器
    container := due.NewContainer()
    // 创建服务注册
    registry := consul.NewRegistry()
    // 创建节点组件
    component := node.NewNode(
        node.WithRegistry(registry),
    )
    // 添加节点组件
    container.Add(component)
    // 启动容器
    container.Serve()
}
```

## 核心 API

### 注册中心接口

```go
type Registry interface {
    // 注册服务
    Register(ctx context.Context, info *ServiceInfo) error
    // 注销服务
    Deregister(ctx context.Context, serviceID string) error
    // 获取服务
    Services(ctx context.Context, serviceName string) ([]*ServiceInfo, error)
    // 监听服务变化
    Watch(ctx context.Context, serviceName string) (Watcher, error)
}
```

### 服务信息结构

```go
type ServiceInfo struct {
    ID       string            // 服务ID
    Name     string            // 服务名称
    Address  string            // 服务地址
    Port     int               // 服务端口
    Metadata map[string]string // 元数据
}
```

### Watcher 接口

```go
type Watcher interface {
    // 获取下一个服务变更事件
    Next() ([]*ServiceInfo, error)
    // 停止监听
    Stop()
}
```

## 服务注册示例

```go
package main

import (
    "context"
    "github.com/dobyte/due/registry/consul/v2"
)

func main() {
    // 创建注册中心
    registry := consul.NewRegistry()
    
    // 注册服务
    err := registry.Register(context.Background(), &consul.ServiceInfo{
        ID:      "service-1",
        Name:    "my-service",
        Address: "127.0.0.1",
        Port:    8080,
        Metadata: map[string]string{
            "version": "v1.0.0",
        },
    })
    if err != nil {
        panic(err)
    }
    
    // 获取服务列表
    services, err := registry.Services(context.Background(), "my-service")
    if err != nil {
        panic(err)
    }
    
    // 监听服务变化
    watcher, err := registry.Watch(context.Background(), "my-service")
    if err != nil {
        panic(err)
    }
    defer watcher.Stop()
    
    for {
        services, err := watcher.Next()
        if err != nil {
            break
        }
        // 处理服务变更
        handleServiceChange(services)
    }
}
```

## 最佳实践

1. **服务命名**:
   - 使用清晰的服务名称便于识别
   - 服务ID应全局唯一

2. **健康检查**:
   - 启用健康检查确保服务可用性
   - 合理设置检查间隔和超时时间

3. **元数据管理**:
   - 控制元数据数量(建议不超过20个)
   - 键长度不超过127字符,值长度不超过512字符

4. **高可用**:
   - 配置自动注销时间防止僵尸服务
   - 使用心跳机制保持服务活性
