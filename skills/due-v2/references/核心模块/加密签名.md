# 加密签名 (Crypto)

## 概述

加密签名模块是 Due 框架的核心组件,提供数据加密、解密、签名、验签等安全能力。支持 RSA、AES 等主流加密算法,保障数据传输安全。

## 包路径

```go
import "github.com/dobyte/due/v2/core/crypto"
```

## RSA 加密

### 配置参数

```toml
[crypto.rsa]
    # 私钥文件路径
    privateKey = "./etc/rsa/private.pem"
    # 公钥文件路径
    publicKey = "./etc/rsa/public.pem"
```

### 使用示例

```go
package main

import (
    "github.com/dobyte/due/v2/core/crypto"
)

func main() {
    // 创建RSA加密器
    encryptor := crypto.NewRSAEncryptor(
        crypto.WithPrivateKey("./etc/rsa/private.pem"),
        crypto.WithPublicKey("./etc/rsa/public.pem"),
    )
    
    // 加密数据
    ciphertext, err := encryptor.Encrypt([]byte("hello world"))
    if err != nil {
        panic(err)
    }
    
    // 解密数据
    plaintext, err := encryptor.Decrypt(ciphertext)
    if err != nil {
        panic(err)
    }
    
    println(string(plaintext))
}
```

## AES 加密

### 使用示例

```go
package main

import (
    "github.com/dobyte/due/v2/core/crypto"
)

func main() {
    // 创建AES加密器

        panic(err)
    }
    
    // 解密数据
    plaintext, err := encryptor.Decrypt(ciphertext)
    if err != nil {
        panic(err)
    }
    
    println(string(plaintext))
}
```

## 数字签名

### RSA 签名示例

```go
package main

import (
    "github.com/dobyte/due/v2/core/crypto"
)

func main() {
    // 创建RSA签名器
    signer := crypto.NewRSASigner(
        crypto.WithPrivateKey("./etc/rsa/private.pem"),
        crypto.WithPublicKey("./etc/rsa/public.pem"),
    )
    
    data := []byte("hello world")
    
    // 签名
    signature, err := signer.Sign(data)
    if err != nil {
        panic(err)
    }
    
    // 验签
    valid := signer.Verify(data, signature)
    if valid {
        println("Signature is valid")
    } else {
        println("Signature is invalid")
    }
}
```

## 核心 API

### 加密器接口

```go
type Encryptor interface {
    // 加密数据
    Encrypt(plaintext []byte) ([]byte, error)
    // 解密数据
    Decrypt(ciphertext []byte) ([]byte, error)
}
```

### 签名器接口

```go
type Signer interface {
    // 签名
    Sign(data []byte) ([]byte, error)
    // 验签
    Verify(data []byte, signature []byte) bool
}
```

## 在网关服务器中使用加密

    "github.com/dobyte/due/v2/cluster/gate"

## 最佳实践

1. **密钥管理**:
   - 使用安全的密钥存储方案
   - 定期轮换密钥
   - 不要将密钥硬编码在代码中

2. **算法选择**:
   - RSA适用于非对称加密和数字签名
   - AES适用于大量数据的对称加密
   - 根据安全需求选择合适的密钥长度

3. **性能优化**:
   - RSA加密速度较慢,适合加密小数据
   - 对于大数据,使用AES加密数据,RSA加密AES密钥

4. **安全性**:
   - 使用足够长的密钥(RSA建议2048位以上)
   - 启用TLS保护传输层
   - 定期更新加密库版本
