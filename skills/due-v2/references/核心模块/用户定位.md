# 用户定位 (Locate)

## 概述

用户定位模块是 Due 框架的核心组件,提供分布式环境下的用户位置追踪能力。支持 Redis、Memcache 等存储后端,用于追踪用户当前所在的服务器节点。

## 包路径

```go
import "github.com/dobyte/due/locate/redis/v2"
```

## Redis 定位器

### 配置参数

```toml
[locate.redis]
    # 客户端连接地址
    addrs = ["127.0.0.1:6379"]
    # 数据库号
    db = 0
    # 用户名
    username = ""
    # 密码
    password = ""
    # 私钥文件
    keyFile = ""
    # 证书文件
    certFile = ""
    # CA证书文件
    caFile = ""
    # 最大重试次数
    maxRetries = 3
    # key前缀
    prefix = "due:locate"
```

### 使用示例

```go
package main

import (
    "github.com/dobyte/due/locate/redis/v2"
    "github.com/dobyte/due/v2/cluster/node"
)

func main() {
    // 创建用户定位器
    locator := redis.NewLocator()
    // 创建节点组件
    component := node.NewNode(
        node.WithLocator(locator),
    )
    // ...
}
```

## 核心 API

### 定位器接口

```go
type Locator interface {
    // 绑定用户和服务器
    Bind(ctx context.Context, uid int64, server string) error
    // 解绑用户
    Unbind(ctx context.Context, uid int64) error
    // 获取用户所在服务器
    Locate(ctx context.Context, uid int64) (string, error)
}
```

### Redis 定位器实现

```go
type Locator struct {
    // ...

func NewLocator(opts ...Option) *Locator

```

## 使用场景示例

```go
package main

import (
    "context"
    "github.com/dobyte/due/locate/redis/v2"
)

func main() {
    // 创建定位器
    locator := redis.NewLocator()
    
    ctx := context.Background()
    
    // 用户登录时绑定到当前服务器
    err := locator.Bind(ctx, 1001, "node-1:8080")
    if err != nil {
        panic(err)
    }
    
    // 查找用户所在服务器
    server, err := locator.Locate(ctx, 1001)
    if err != nil {
        panic(err)
    }
    println("User located at:", server)
    
    // 用户登出时解绑
    err = locator.Unbind(ctx, 1001)
    if err != nil {
        panic(err)
    }
}
```

## 最佳实践

1. **Key前缀**:
   - 使用清晰的key前缀避免冲突
   - 默认前缀为 `due:locate`

2. **连接池配置**:
   - 合理设置最大连接数
   - 配置连接超时时间

3. **重试机制**:
   - 设置最大重试次数提高可用性
   - 处理网络抖动情况

4. **清理策略**:
   - 用户登出时及时解绑
   - 定期清理过期数据
