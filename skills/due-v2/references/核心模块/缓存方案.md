# 缓存方案 (Cache)

## 概述

缓存方案是 Due 框架的扩展组件，提供高性能的缓存管理能力。支持 Redis、Memcache 等存储后端，用于加速数据访问和减轻数据库压力。

## 包路径

```go
import "github.com/dobyte/due/cache/redis/v2"
```

## Redis 缓存

### 配置参数

```toml
[cache.redis]
    # 客户端连接地址
    addrs = ["127.0.0.1:6379"]
    # 数据库号
    db = 0
    # 用户名
    username = ""
    # 密码
    password = ""
    # 私钥文件
    keyFile = ""
    # 证书文件
    certFile = ""
    # CA证书文件
    caFile = ""
    # 最大重试次数
    maxRetries = 3
    # key前缀
    prefix = "due:cache"
```

### 使用示例

```go
package main

import (
    "github.com/dobyte/due/cache/redis/v2"
    "github.com/dobyte/due/v2"
)

func main() {
    // 创建容器
    container := due.NewContainer()
    // 创建缓存
    cache := redis.NewCache()
    // 添加到容器
    container.Add(cache)
    // 启动容器
    container.Serve()
}
```

## 核心 API

### 缓存接口

```go
type Cache interface {
    // 获取缓存值
    Get(ctx context.Context, key string, value interface{}) error
    // 设置缓存值
    Set(ctx context.Context, key string, value interface{}, ttl time.Duration) error
    // 删除缓存
    Delete(ctx context.Context, keys ...string) error
    // 检查缓存是否存在
    Exists(ctx context.Context, key string) (bool, error)
    // 清空所有缓存
    Clear(ctx context.Context) error
}
```

## 使用场景示例

### 基础使用

```go
package main

import (
    "context"
    "time"
    "github.com/dobyte/due/cache/redis/v2"
)

func main() {
    // 创建缓存
    cache := redis.NewCache()
    
    ctx := context.Background()
    
    // 设置缓存
    err := cache.Set(ctx, "user:1001", map[string]interface{}{
        "id":   1001,
        "name": "Alice",
    }, 10*time.Minute)
    if err != nil {
        panic(err)
    }
    
    // 获取缓存
    var user map[string]interface{}
    err = cache.Get(ctx, "user:1001", &user)
    if err != nil {
        panic(err)
    }
    
    println(user["name"].(string))
    
    // 删除缓存
    err = cache.Delete(ctx, "user:1001")
    if err != nil {
        panic(err)
    }
}
```

### 缓存穿透防护

```go
package main

import (
    "context"
    "time"
    "github.com/dobyte/due/cache/redis/v2"
)

func getUser(ctx context.Context, cache *redis.Cache, userID int64) (*User, error) {
    key := fmt.Sprintf("user:%d", userID)
    
    // 尝试从缓存获取
    var user User
    err := cache.Get(ctx, key, &user)
    if err == nil {
        return &user, nil
    }
    
    // 缓存未命中，查询数据库
    user, err = queryUserFromDB(ctx, userID)
    if err != nil {
        return nil, err
    }
    
    // 写入缓存
    err = cache.Set(ctx, key, user, 10*time.Minute)
    if err != nil {
        // 记录日志但不影响业务
        log.Warnf("Failed to set cache: %v", err)
    }
    
    return &user, nil
}
```

### 批量操作

```go
package main

import (
    "context"
    "github.com/dobyte/due/cache/redis/v2"
)

func batchDeleteCache(ctx context.Context, cache *redis.Cache, keys []string) error {
    // 批量删除缓存
    return cache.Delete(ctx, keys...)
}
```

## 最佳实践

1. **缓存策略**:
   - 使用合理的过期时间避免数据不一致
   - 对于热点数据设置较长的缓存时间

2. **缓存穿透**:
   - 对于空值也进行缓存（设置较短过期时间）
   - 使用布隆过滤器预防缓存穿透

3. **缓存雪崩**:
   - 为缓存添加随机过期时间
   - 使用多级缓存架构

4. **Key命名**:
   - 使用清晰的key前缀避免冲突
   - 包含业务标识便于管理和调试

5. **序列化**:
   - 选择合适的序列化格式（JSON、Protobuf等）
   - 考虑序列化性能对整体性能的影响
