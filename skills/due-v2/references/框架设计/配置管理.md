# 配置管理

## 概述

Due v2 使用 TOML 格式的配置文件，支持多层级配置、环境变量覆盖、配置热加载等特性。配置系统分为两层：
- **etc**：框架启动配置，用于初始化核心组件
- **config**：业务配置，用于游戏逻辑参数

## 配置文件结构

### 目录结构

```
项目根目录/
├── etc/
│   ├── etc.toml          # 框架启动配置（必需）
│   └── etc.local.toml    # 本地覆盖配置（可选）
├── config/
│   ├── game.toml         # 游戏业务配置
│   └── game.local.toml   # 本地覆盖配置（可选）
└── main.go
```

### 配置优先级

配置加载顺序（后加载的覆盖先加载的）：

```
1. etc.toml（基础配置）
   ↓
2. etc.local.toml（本地覆盖）
   ↓
3. 环境变量（最高优先级）
```

## 启动配置（etc）

### 核心配置项

#### 1. 进程配置

```toml
# 进程 ID 文件路径
pid = "./run/due.pid"

# 时区设置
timezone = "Asia/Shanghai"

# 关闭超时时间
shutdownMaxWaitTime = "30s"
```

#### 2. 运行模式

```toml
# 运行模式：debug、test、release
mode = "debug"
```

**环境变量覆盖：**
```bash
export DUE_MODE=release
```

#### 3. 网关服配置

```toml
[cluster.gate]
  # 网关服 ID（留空则自动生成）
  id = ""
  
  # 网关服名称
  name = "gate"
  
  # RPC 监听地址
  addr = ":8001"
  
  # 是否暴露 RPC 地址到公网
  expose = false
  
  # RPC 超时时间
  timeout = "3s"
  
  # 无状态路由分发策略：random、round-robin、consistent-hash
  dispatch = "random"
```

#### 4. 节点服配置

```toml
[cluster.node]
  # 节点服 ID（留空则自动生成）
  id = ""
  
  # 节点服名称
  name = "node"
  
  # RPC 监听地址
  addr = ":8002"
  
  # 是否暴露 RPC 地址到公网
  expose = false
  
  # 消息编解码器：json、proto
  codec = "proto"
  
  # RPC 超时时间
  timeout = "3s"
  
  # 节点权重（用于加权轮询）
  weight = 1
```

#### 5. 网格服配置

```toml
[cluster.mesh]
  # 网格服 ID
  id = ""
  
  # 网格服名称
  name = "mesh"
  
  # 是否启用网格服
  enable = false
  
  # RPC 监听地址
  addr = ":8003"
  
  # 消息编解码器
  codec = "proto"
```

#### 6. 协议配置

```toml
[packet]
  # 路由号字节数：1、2、4
  routeBytes = 2
  
  # 序列号字节数：0、1、2、4
  seqBytes = 2
  
  # 消息数据最大字节数
  bufferBytes = 5000
  
  # 心跳包是否携带时间戳
  heartbeatTime = false

#### 7. 网络配置

```

## 业务配置（config）

### 配置读取

```go
import "github.com/dobyte/due/v2/config"

// 读取单个配置项
maxLevel := config.Get("game.maxLevel", 100).Int()

// 读取配置到结构体
type GameConfig struct {
    MaxLevel    int     `json:"maxLevel"`
    ExpRate     float64 `json:"expRate"`
    DropRate    float64 `json:"dropRate"`
}

gameConfig := &GameConfig{}
if err := config.Get("game").Scan(gameConfig); err != nil {
    log.Errorf("读取游戏配置失败: %v", err)
}
```

### 配置文件示例

```toml
# config/game.toml

[game]
  # 最大等级
  maxLevel = 100
  
  # 经验倍率
  expRate = 1.5
  
  # 掉落倍率
  dropRate = 1.2

[game.battle]
  # 战斗超时时间
  timeout = 300  # 秒
  
  # 最大参战人数
  maxPlayers = 5

[game.shop]
  # 商城刷新间隔
  refreshInterval = 3600  # 秒
  
  # 商品列表
  [[game.shop.items]]
    id = 1001
    name = "生命药水"
    price = 100
    
  [[game.shop.items]]
    id = 1002
    name = "魔法药水"
    price = 150
```

## 环境变量覆盖

### 支持的配置项

```bash
# 运行模式
export DUE_MODE=release

# 网关服地址
export DUE_CLUSTER_GATE_ADDR=:9001

# 节点服地址
export DUE_CLUSTER_NODE_ADDR=:9002

# Redis 地址
export DUE_LOCATE_REDIS_ADDRS=192.168.1.100:6379

# 日志级别
export DUE_LOG_LEVEL=error
```

### 环境变量命名规则

```
DUE_{SECTION}_{KEY}

示例：
[cluster.gate]
  addr = ":8001"
  
对应环境变量：
DUE_CLUSTER_GATE_ADDR=:9001
```

## 配置最佳实践

### 1. 分离环境配置

```
etc/
├── etc.toml              # 通用配置
├── etc.dev.toml          # 开发环境
├── etc.test.toml         # 测试环境
└── etc.prod.toml         # 生产环境
```

**启动时指定环境：**
```bash
# 开发环境
DUE_ENV=dev ./game-server

# 生产环境
DUE_ENV=prod ./game-server
```

### 2. 敏感信息使用环境变量

```toml
# etc.toml
[locate.redis]
  addrs = ["127.0.0.1:6379"]
  password = ""  # 不在配置文件中写密码
```

```bash
# 启动时通过环境变量传入
export DUE_LOCATE_REDIS_PASSWORD=your_secure_password
./game-server
```

### 3. 配置默认值

```go
// 读取配置时提供默认值
maxLevel := config.Get("game.maxLevel", 100).Int()
timeout := config.Get("game.battle.timeout", 300).Int()
```

### 4. 配置验证

```go
func validateConfig() error {
    maxLevel := config.Get("game.maxLevel", 0).Int()
    if maxLevel <= 0 {
        return errors.New("game.maxLevel 必须大于 0")
    }
    
    expRate := config.Get("game.expRate", 0).Float64()
    if expRate < 0 {
        return errors.New("game.expRate 不能为负数")
    }
    
    return nil
}
```

## 常见问题

### Q1: 配置文件找不到怎么办？

**A:** 确保配置文件在正确的位置：
- 启动配置：`./etc/etc.toml`
- 业务配置：`./config/*.toml`

可以通过环境变量指定路径：
```bash
export DUE_ETC_DIR=/path/to/etc
export DUE_CONFIG_DIR=/path/to/config
```

### Q2: 配置热加载不生效？

**A:** 只有 `config` 目录下的业务配置支持热加载，`etc` 目录下的启动配置不支持热加载。

### Q3: 如何查看所有配置项？

**A:** 使用调试模式启动，日志会输出所有加载的配置项：
```bash
DUE_MODE=debug ./game-server
```

### Q4: 配置项类型转换失败？

**A:** 确保配置值类型正确：
```toml
# 错误：字符串
maxLevel = "100"

# 正确：整数
maxLevel = 100
```
