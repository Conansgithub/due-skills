# 错误处理

## 概述

Go 标准库的 `error` 相对简单，无法传递错误的状态信息。Due v2 框架专门设计了一套错误处理机制，通过 `errors.NewError()` 和 `codes.Convert()` 实现错误在集群服务间的自由转换和传递。

为避免与标准库 `errors` 命名冲突，框架库 `errors` 重载了标准库 `errors` 的所有函数。无论创建普通 error 还是框架 error，都可以统一使用框架库的 `errors`。

## 核心概念

### 错误码（Code）

错误码包含数字码和描述信息，可在集群服务间传递：

```go
// 创建自定义错误码
code := codes.NewCode(404, "not found")

// 内置错误码
codes.OK           // 成功
codes.InternalError // 内部错误
codes.InvalidArgument // 无效参数
codes.Unauthorized    // 未授权
codes.NotFound        // 未找到
codes.IllegalRequest  // 非法请求
```

### 框架错误（Error）

通过 `errors.NewError()` 创建携带错误码的 error：

```go
err := errors.NewError(codes.NewCode(404, "not found"))
```

### 错误转换

通过 `codes.Convert()` 从 error 中提取错误码：

```go
code := codes.Convert(err)
```

## 使用方式

### 创建错误

```go
// 使用内置错误码
err := errors.NewError(codes.InternalError)

// 使用自定义错误码
err := errors.NewError(codes.NewCode(404, "not found"))

// 附带原始错误
err := errors.NewError(err, codes.InternalError)

// 标准错误（兼容标准库）
err := errors.New("something went wrong")
```

### 转换与提取错误码

```go
// 从 error 中提取错误码
code := codes.Convert(err)

// 获取错误码数值
codeNum := code.Code()    // 返回 int

// 获取错误描述
msg := code.Message()     // 返回 string
```

### 在路由处理器中使用

```go
func handler(ctx node.Context) {
    res := &myRes{}
    ctx.Defer(func() {
        ctx.Response(res)
    })

    // 业务逻辑出错时设置错误码
    if err := doSomething(); err != nil {
        res.Code = codes.Convert(err).Code()
        return
    }

    res.Code = codes.OK.Code()
}
```

### 在业务层返回错误

```go
func (c *core) doLogin(req *loginReq) (int64, error) {
    if req.Password != defaultPassword {
        return 0, errors.NewError(codes.InvalidArgument)
    }
    // ...
    return uid, nil
}

// 调用处转换
uid, err := c.doLogin(req)
if err != nil {
    res.Code = codes.Convert(err).Code()
    return
}
```

## 完整示例

```go
package main

import (
    "github.com/dobyte/due/v2/codes"
    "github.com/dobyte/due/v2/errors"
    "github.com/dobyte/due/v2/log"
)

func main() {
    err := errors.NewError(codes.NewCode(404, "not found"))

    code := codes.Convert(err)

    log.Infof("error: %s", err.Error())
    log.Infof("code: %d", code.Code())
    log.Infof("message: %s", code.Message())
}
```

### 运行结果

```

INFO[2025/10/31 14:29:23.610934] main.go:15 code: 404
INFO[2025/10/31 14:29:23.610934] main.go:16 message: not found
```

## 集群间错误传递

框架错误处理机制的核心优势在于支持错误在集群服务间传递：

```
客户端 → Gate → Node → Mesh
         ↓        ↓       ↓
       error    error   error  (错误码在各层间无损传递)
```

1. Node 层业务逻辑产生错误：`errors.NewError(codes.NotFound)`
2. 通过 RPC 调用传递到 Gate 层
3. Gate 层通过 `codes.Convert(err)` 提取错误码
4. 将错误码数值返回给客户端

## 与标准库的兼容性

框架 `errors` 包完全兼容标准库 `errors` 的所有函数：

```go
import "github.com/dobyte/due/v2/errors"

// 以下函数与标准库 errors 完全一致
errors.New("message")
errors.Is(err, target)
errors.As(err, target)
errors.Unwrap(err)
```

## 最佳实践

1. **统一使用框架 errors**：避免同时导入标准库和框架库的 errors
2. **业务层返回框架错误**：使用 `errors.NewError(code)` 携带错误码
3. **处理器中转换**：使用 `codes.Convert(err).Code()` 提取数值错误码
4. **自定义错误码**：使用 `codes.NewCode()` 定义业务专属错误码
5. **错误码规范**：建议统一管理错误码定义，避免码值冲突
