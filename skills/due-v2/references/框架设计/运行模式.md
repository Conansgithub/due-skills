# 运行模式

## 概述

Due v2 框架设计了 debug、test、release 三种运行模式，通过外部参数控制程序内部预先定义的行为。开发者可以根据需要合理选择运行模式，在不同环境下调整程序表现。

## 三种模式

| 模式 | 说明 | 典型用途 |
|------|------|---------|
| `debug` | 调试模式 | 本地开发调试，输出详细日志 |
| `test` | 测试模式 | 自动化测试环境 |
| `release` | 发布模式 | 生产环境部署 |

## 设置方式

支持四种方式设置运行模式，优先级从低到高：

### 1. 配置文件

在 `etc/etc.toml` 中设置：

```toml
mode = "debug"
```

### 2. 环境变量

```bash
export DUE_MODE=debug
```

### 3. 启动参数

```bash
go run main.go --mode=debug
```

### 4. 代码调用

```go
import "github.com/dobyte/due/v2/mode"

mode.SetMode("debug")
```

### 优先级规则

```
配置文件 < 环境变量 < 运行参数 < mode.SetMode()
```

即 `mode.SetMode()` 优先级最高，配置文件优先级最低。

## 相关接口

### 导入包

```go
import "github.com/dobyte/due/v2/mode"
```

### 常用函数

| 函数 | 返回值 | 说明 |
|------|--------|------|
| `mode.GetMode()` | `string` | 获取当前运行模式 |
| `mode.IsDebugMode()` | `bool` | 是否为 debug 模式 |
| `mode.IsTestMode()` | `bool` | 是否为 test 模式 |
| `mode.IsReleaseMode()` | `bool` | 是否为 release 模式 |

### API 文档

[https://pkg.go.dev/github.com/dobyte/due/v2/mode](https://pkg.go.dev/github.com/dobyte/due/v2/mode)

## 使用示例

```go
package main

import (
    "github.com/dobyte/due/v2/log"
    "github.com/dobyte/due/v2/mode"
)

func main() {
    log.Infof("mode: %s", mode.GetMode())
    log.Infof("debug mode: %v", mode.IsDebugMode())
    log.Infof("test mode: %v", mode.IsTestMode())
    log.Infof("release mode: %v", mode.IsReleaseMode())
}
```

### 运行结果

```bash
$ go run main.go
INFO[2025/10/31 14:26:18.361636] main.go:9 mode: debug
INFO[2025/10/31 14:26:18.362173] main.go:10 debug mode: true
INFO[2025/10/31 14:26:18.362173] main.go:11 test mode: false
INFO[2025/10/31 14:26:18.362173] main.go:12 release mode: false
```

## 常见应用场景

### 条件性日志输出

```go
if mode.IsDebugMode() {
    log.Debugf("详细调试信息: %v", data)
}
```

### 条件性功能开关

```go
if mode.IsReleaseMode() {
    // 生产环境启用性能监控
    enableProfiling()
}
```

### 环境差异化配置

```go
switch mode.GetMode() {
case "debug":
    dbConfig.MaxOpenConns = 5
case "test":
    dbConfig.MaxOpenConns = 10
case "release":
    dbConfig.MaxOpenConns = 100
}
```

## 注意事项

1. **提前设置**：`mode.SetMode()` 应在程序初始化早期调用，最好在 `due.NewContainer()` 之前
2. **一致性**：集群中各节点应使用相同的运行模式
3. **安全**：release 模式下避免输出敏感调试信息
